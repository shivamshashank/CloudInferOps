StackPulse is a ONE-COMMAND CLI for deploying production-grade observability
stacks on AWS. It handles everything:

  🚀 One-tap deployment     → Deploys in single command
  📊 Metrics (Prometheus)   → Metrics collection & storage
  📜 Logs (Loki)            → Log aggregation
  🔍 Traces (Tempo)         → Distributed tracing
  📈 Dashboards (Grafana)   → Beautiful visualizations
  🚨 Alerts (AlertManager)  → Intelligent routing
  🔔 Integrations           → Slack, PagerDuty
  🔁 GitOps (ArgoCD)        → Declarative deployments
  ☁️  AWS-native            → EKS & EC2 support

⚡ QUICK START
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Configure AWS:
   $ aws configure

2. Deploy to EKS:
   $ ./stackpulse up --region eu-west-2 --cluster-name my-cluster

3. Deploy to EC2:
   $ ./stackpulse up --region us-east-1 --cluster-name my-ec2-instance

4. Configure Alerts:
   $ ./stackpulse integrate slack --webhook-url YOUR_WEBHOOK_URL

5. Access URLs (provided after deployment):
   • Grafana:       http://grafana.YOUR_IP.nip.io (admin/admin)
   • Prometheus:    http://prometheus.YOUR_IP.nip.io
   • AlertManager:  http://alertmanager.YOUR_IP.nip.io
   • PushGateway:   http://pushgateway.YOUR_IP.nip.io

🎯 DEPLOYMENT FLOWS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

EKS Flow:
  1. Verify AWS credentials
  2. Check for existing EKS cluster by name
  3. Update local kubeconfig
  4. Deploy kube-prometheus-stack via Helm
  5. Deploy Grafana via Helm
  6. Configure ingress & DNS
  7. Display access URLs

EC2 Flow:
  1. Verify AWS credentials
  2. Check for existing EC2 instance by name tag
  3. Attach IAM SSM role (auto-create if needed)
  4. Send SSM commands to instance:
     - Check Kubernetes status (K3s)
     - Install/update Helm
     - Deploy NGINX Ingress
     - Deploy Prometheus stack
     - Deploy Grafana
  5. Display access URLs with nip.io DNS
